١ المتطلبات الأساسية (Prerequisites)

البرنامج الإصدار المطلوب رابط التحميل الاستخدام
Docker Desktop 4.x أو أحدث docker.com/products/docker-desktop تشغيل جميع الخدمات
Git 2.x أو أحدث git-scm.com استنساخ المشروع
Node.js 18 أو أحدث (اختياري) nodejs.org للتطوير المحلي فقط
⚠️ تأكد أن Docker Desktop يعمل قبل المتابعة افتح Docker Desktop وانتظر حتى تظهر أيقونة البدء الخضراء في شريط المهام.

٢ استنساخ المشروع من GitHub

🔗 رابط المستودع

REPOSITORY 
https://github.com/aliqaisy62-ux/ims-pro.git
Branch: android-files

📥 أوامر الاستنساخ والإعداد

TERMINAL / CMD / POWERSHELL 
# 1. استنساخ المشروع
git clone https://github.com/aliqaisy62-ux/ims-pro.git
cd ims-pro

# 2. نسخ ملف الإعدادات
cp .env.example .env
cp .env.example .env.docker
# Windows:
copy .env.example .env
copy .env.example .env.docker

# 3. تعديل JWT_SECRET في ملف .env (مطلوب للإنتاج)
# استبدل GENERATE_A_64_CHAR_RANDOM_STRING_HERE بقيمة عشوائية

🔑 إنشاء JWT Secret آمن (مرة واحدة فقط)

NODE.JS 
# شغّل هذا الأمر مرتين — مرة لكل secret
node -e "console.log(require('crypto').randomBytes(64).toString('base64'))"

# ضع الناتج الأول في:  JWT_SECRET=...
# ضع الناتج الثاني في: JWT_REFRESH_SECRET=...
💡 للتشغيل السريع (Development فقط) يمكنك تخطي إنشاء JWT Secret — الملفات تحتوي على قيم تطوير جاهزة: dev-only-jwt-secret-32-chars-minimum-local — لا تستخدمها في الإنتاج.

٣ تشغيل المشروع

🚀 الطريقة الأولى — الإنتاج (Production) — موصى به

تشغيل كامل النظام عبر Docker بأمر واحد:

TERMINAL — داخل مجلد ims-pro 
# تأكد أن SEED_DB=true في .env (للتشغيل الأول فقط)

# بناء وتشغيل جميع الخدمات
docker-compose up --build -d

# مشاهدة حالة الخدمات
docker ps

# مشاهدة السجلات (اختياري)
docker-compose logs -f
✅ وقت التشغيل الأول يستغرق البناء الأول 3–8 دقائق بحسب سرعة الإنترنت (يحمّل Docker Images). الإطلاقات التالية تستغرق أقل من 30 ثانية.

🛠️ الطريقة الثانية — التطوير (Development)

TERMINAL 
# تشغيل بيئة التطوير مع Hot-Reload
docker-compose -f docker-compose.dev.yml up --build -d

# API على:  http://localhost:4001
# Web على:  http://localhost:3001

✅ التحقق من تشغيل الخدمات

TERMINAL 
# عرض حالة جميع الـ Containers
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# يجب أن ترى هذه الخدمات بحالة "healthy":
ims-pro-db     Up (healthy)   0.0.0.0:5434->5432/tcp
ims-pro-api    Up (healthy)   0.0.0.0:4000->4000/tcp
ims-pro-web    Up (healthy)   0.0.0.0:3002->3000/tcp
ims-pro-nginx  Up (healthy)   0.0.0.0:80->80/tcp

🔄 أوامر إدارية مفيدة

TERMINAL 
# إيقاف جميع الخدمات
docker-compose down

# إعادة تشغيل خدمة واحدة
docker-compose restart api

# إعادة البناء بعد تغيير الكود
docker-compose up --build -d

# عرض سجلات API
docker logs ims-pro-api -f --tail 50

# حذف كل شيء والبدء من جديد (يحذف البيانات!)
docker-compose down -v

٦ خطوات الإعداد الأول (First-Time Setup)

١

افتح التطبيق في المتصفح

اذهب إلى http://localhost أو http://localhost:3001

٢

سجّل دخولك كمدير النظام

استخدم: admin@system.com / Admin@12345

٣

اضبط سعر الصرف

الإعدادات ← سعر الصرف ← أدخل سعر الدولار الحالي بالدينار العراقي

٤

أضف المنتجات

المنتجات ← إضافة منتج ← أدخل الباركود والأسعار الخمسة والمخزون الابتدائي

٥

أنشئ حسابات المستخدمين

الإعدادات ← إدارة المستخدمين ← أضف كاشيرين وموظفين بكلمات مرورهم

٦

جرّب نقطة البيع

اذهب إلى نقطة البيع ← امسح باركود منتج ← اختر طريقة الدفع ← أتمم البيع

٧

مزامنة قاعدة البيانات (بعد تحديث النظام)

الإعدادات ← صيانة النظام ← "مزامنة قاعدة البيانات" لتطبيق أي تحديثات
